# プラズマシミュレーター開発用 自作PC 構成指示書

**目的:** 設計提案書 v1.0–v2.0 の Phase 0–2 を実機で実行・検証できる最低予算のPC  
**購入場所:** 秋葉原（新品 + 中古の併用を前提）  
**作成日:** 2026-04-02

---

## 0. 設計思想：何に金をかけ、何を削るか

このPCの目的はゲームではない。**JAX + CUDA でプラズマPICシミュレーションを回し、PINOを学習させ、微分可能シミュレーションの勾配を取る**ことである。

ワークロードの特性を踏まえた優先順位：

| 優先度 | 部品 | 理由 |
|-------|------|------|
| **最優先** | GPU (VRAM量) | PIC粒子データ + PINO推論 + jax.checkpoint の中間状態がすべてVRAMに乗る |
| **高** | RAM (容量) | WarpXデータの前処理、PINO学習データのバッファ、AMMC Layer Aのメッシュ管理 |
| **中** | SSD (速度) | チェックポイントI/O、HDF5データの読み書き |
| **低** | CPU (コア数) | JAXはGPUで計算。CPUはオーケストレーション + AMMC制御のみ |
| **最低** | ケース・見た目 | どうでもいい |

**最重要原則：1円でも多くGPUのVRAMに回せ。**

## 1. 各フェーズで必要な計算資源の見積もり

まず「何を動かすのか」から逆算する：

### Phase 0（基盤整備）— このPCで完遂する

| タスク | GPU VRAM | RAM | 計算時間目安 |
|--------|----------|-----|------------|
| 1D3V PIC (JAX-in-Cell) Landau減衰 | ~1 GB | ~4 GB | 数分 |
| 1D Vlasov-Poisson 二流体不安定性 | ~2 GB | ~4 GB | 数分 |
| FNO/DeepONet 1D問題の学習 | ~2-4 GB | ~8 GB | 数十分 |
| バケットシステム設計検証 | ~1 GB | ~4 GB | 秒単位 |

### Phase 1（MVP — 微分可能な静的結合）— このPCで実行する

| タスク | GPU VRAM | RAM | 計算時間目安 |
|--------|----------|-----|------------|
| 2D3V PIC (128² grid, PPC=100) | ~4-6 GB | ~8 GB | 数時間 |
| 2D MHD (HLLC, 256² grid) | ~1-2 GB | ~4 GB | 数十分 |
| PIC↔MHD 静的結合 + GTL | ~6-8 GB | ~16 GB | 数時間 |
| End-to-end AD検証 (jax.checkpoint) | ~8-12 GB | ~16 GB | 数時間〜半日 |
| GEM磁気再結合 (2D) | ~8-12 GB | ~16 GB | 半日〜1日 |

### Phase 2（動的結合 + PINO）— このPCでプロトタイプ、本番は要追加投資

| タスク | GPU VRAM | RAM | 計算時間目安 |
|--------|----------|-----|------------|
| AMMC動的セル再分類 (2D) | ~8-12 GB | ~32 GB | 数時間 |
| PINO (2D Vlasov) 学習 | ~12-16 GB | ~32 GB | 数日 |
| PINO + SympMat補正の検証 | ~10-14 GB | ~32 GB | 1日 |
| Kelvin-Helmholtz不安定性 (2D) | ~12-20 GB | ~32 GB | 1-3日 |

### ボトルネック分析

**VRAMが律速**。Phase 1完遂には最低12 GB、Phase 2のプロトタイプには16 GB以上が必要。24 GBあればPhase 2をほぼ完全にカバーできる。

### 結論

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   最低予算で Phase 2 まで完遂する唯一の合理的選択：          │
│                                                             │

│   • VRAM/円 で RTX 4060 Ti 16GB より優位                    │
│   • FP32: 35.6 TFLOPS（科学計算に十分）                     │
│   • 欠点：消費電力350W、2スロット超の巨体、ファン音          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**秋葉原での入手先候補：**
- じゃんぱら（各店舗）— 中古GPU在庫が豊富。動作保証あり
- ソフマップ リユース総合館 — 保証付き中古
- ドスパラ 中古 — 在庫あれば安い
- ツクモ — 中古コーナーが穴場
- イオシス — 掘り出し物があることも

**購入時の注意：**
- マイニング落ちに注意。外観がきれいでファンが異音ならパス
- 可能なら店頭でGPU-Zの表示を確認（VRAM 24GB, バス幅 384bit であること）
- Founders Edition よりサードパーティ（EVGA, MSI, ASUS）の方が冷却が良い
- 保証期間を確認（最低1ヶ月はほしい）

**予算が厳しい場合の妥協案：**
- 中古 RTX 3060 12GB（¥25,000-35,000）で Phase 1 まで進め、Phase 2 で GPU を換装する
- ただしGPU換装時に電源も交換が必要になる可能性あり（3060: 170W → 3090: 350W）

## 4. なぜこの構成なのか — 各パーツの選定根拠

### CPU: Ryzen 5 7600 で十分な理由

JAXの物理計算はほぼ全てGPUで実行される。CPUが担当するのは：
- Layer A（AMMCメッシュ管理）: シングルスレッド性能で十分
- Rayオーケストレーション: 軽量
- データ前処理: メモリ帯域律速（CPUコア数ではなくRAM速度が効く）

Ryzen 7 7700 (+¥10,000-15,000) にしてもシミュレーション速度はほぼ変わらない。その¥15,000をGPUかRAMに回すべき。

### RAM: 64GB を推奨する理由

Phase 2 での典型的なメモリ使用パターン：

```
GPU (RTX 3090, 24GB VRAM)          CPU (RAM, 64GB)
┌────────────────────────┐         ┌────────────────────────┐
│ PIC粒子データ   ~8 GB  │         │ WarpXデータバッファ ~16GB│
│ 場データ        ~2 GB  │         │ AMMC メッシュ管理 ~4 GB │
│ PINOモデル      ~2 GB  │         │ PINO学習データ   ~8 GB │
│ jax.checkpoint  ~6 GB  │         │ 診断・可視化    ~4 GB  │
│ JAXワーク領域   ~4 GB  │         │ OS + その他     ~8 GB  │
│ 空き            ~2 GB  │         │ 空き           ~24 GB  │
└────────────────────────┘         └────────────────────────┘
        合計 ~22 GB                        合計 ~40 GB
```

32GBだとスワップが発生し始める。64GBなら余裕がある。

### SSD: 2TB を推奨する理由

- WarpX出力データ（学習用）: 数百GB
- チェックポイント（HDF5）: 数十GB
- PINOモデル + 学習ログ: 数十GB
- OS + 開発環境: ~50GB

1TBでは頻繁にデータの削除・管理が必要になり、研究のフローが悪化する。

### 電源: 850W の理由

RTX 3090 の瞬間的な電力スパイクは公称TDP (350W) を大幅に超えることがある（実測で最大 ~450W のスパイク）。750W電源では保護機能が働いてシャットダウンするリスクがある。850Wは安全マージン。

## 5. 秋葉原での購入ルート

### 推奨ルート（1日で完了）

```
【午前】GPU確保（最重要）
  ↓
  じゃんぱら 各店舗を巡回 → RTX 3090 の在庫・状態・価格を比較
  ソフマップ リユース総合館 → 保証付き在庫を確認
  (GPU は争奪戦。見つけたら即決すること)
  ↓
【昼】新品パーツ一括購入
  ↓
  ツクモ eX. or ドスパラ パーツ館
  → CPU, マザボ, RAM, SSD, 電源, ケースをまとめ買い
  → まとめ買い割引を交渉（¥3,000-5,000引ける場合あり）
  ↓
【午後】組み立て（自宅）
```

### 購入時チェックリスト

- [ ] RTX 3090: GPU-Z確認、ファン異音なし、外観にヤニ・サビなし
- [ ] RTX 3090: 補助電源コネクタは 8pin×2。電源ケーブルが対応しているか確認
- [ ] RTX 3090: 物理サイズを確認（全長 ~313mm）。ケースに入るか
- [ ] マザーボード: PCIe x16 スロットの位置と3090の厚み（3スロット占有）の干渉確認
- [ ] 電源: 8pin PCIe 補助電源ケーブルが2本以上付属しているか
- [ ] RAM: DDR5であること（DDR4と物理的に互換性なし）
- [ ] SSD: M.2 NVMe であること（SATA M.2 と間違えない）

## 6. セットアップ手順（組み立て後）

### OS: Ubuntu 24.04 LTS

```bash
# 1. NVIDIA ドライバ
sudo apt update && sudo apt install -y nvidia-driver-560

# 2. CUDA Toolkit 12.x
# NVIDIA公式サイトから .run または apt repo で導入
# https://developer.nvidia.com/cuda-downloads

# 3. 動作確認
nvidia-smi   # RTX 3090, 24GB VRAM が表示されること

# 4. Python環境（miniconda推奨）
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh

# 5. JAX (GPU版)
pip install --upgrade "jax[cuda12]"

# 6. 動作確認
python3 -c "import jax; print(jax.devices())"
# → [GpuDevice(id=0, process_index=0)]  が表示されること

# 7. 関連ライブラリ
pip install flax equinox optax diffrax \
            matplotlib h5py pyarrow redis \
            wandb ray[default]

# 8. JAX-in-Cell（Phase 0 の最初のタスク）
# リポジトリをクローンして1D Landau減衰を実行
```

### 最初に走らせるべきベンチマーク

```bash
# JAX GPU性能確認
python3 -c "
import jax
import jax.numpy as jnp

# 行列乗算ベンチマーク (VRAM確認を兼ねる)
x = jnp.ones((8192, 8192))  # ~500MB
%timeit jax.block_until_ready(x @ x)

# VRAM使用量確認
print(jax.devices()[0])
"
```

## 7. 将来のアップグレードパス

このPC構成は、Phase 2までの検証後に段階的に強化できる設計にしてある：

| 時期 | アップグレード | 費用目安 | 効果 |
|------|-------------|---------|------|
| Phase 2 で VRAM 不足時 | RTX 3090 → 中古 RTX 3090 Ti (24GB) or RTX 4090 (24GB) | ¥100,000-220,000 | CUDA cores 増、電力効率改善 |
| Phase 3 で RAM 不足時 | 64GB → 128GB (32GB×4) | ¥30,000-40,000 | B650は最大128GB対応 |
| Phase 4 でマルチGPU | 2台目 GPU 追加（要マザボ・電源・ケース交換） | — | 現実的にはクラウド(Lambda, Vast.ai)を検討 |

### Phase 3–4 はクラウドHPCとのハイブリッドが現実的

Phase 4（3Dトロイダル、ITERスケール）を自宅PCで実行するのは不可能。以下のハイブリッド運用を想定：

```
自宅PC (RTX 3090)                    クラウド / HPC
──────────────────                   ──────────────
• コード開発・デバッグ                 • 大規模本番実行
• 2Dプロトタイプ検証                   • 3D GEM再結合
• PINO小規模学習                      • ELMサイクル模擬
• Auto-research骨格テスト              • マルチGPUスケーリング
• 解析・可視化                         • WarpX学習データ生成
     │                                    │
     └──── git push / pull ────────────────┘
           同一コードベース（JAXの利点）
```

**クラウド費用の目安:**
- Lambda Cloud (A100 80GB × 1): ~$1.10/hr → 1日 ~¥4,000
- Vast.ai (RTX 3090 × 1): ~$0.20/hr → 1日 ~¥700
- Google Colab Pro+ (A100): ¥5,000/月（不安定だが最安）
